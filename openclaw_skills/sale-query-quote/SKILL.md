# 查询报价总览 (Sale Query Quote)

该 Skill 用于查询指定客户在指定日期或天数范围内的报价记录关键字段（如日期、报价编号、型号、品牌、数量、批号、货期、备注等）。

## 环境配置

使用前需设置环境变量（可选，有默认值）：

```bash
# Windows (PowerShell)
$env:UNIULTRA_DB_PATH = “E:\WorkPlace\7_AI_APP\UniUltraOpenPlatForm\uni_platform.db”

# Linux/WSL
export UNIULTRA_DB_PATH=”/mnt/e/WorkPlace/7_AI_APP/UniUltraOpenPlatForm/uni_platform.db”
```

详见 `openclaw_skills/ENV_SETUP.md`。

## 工作流程

### 第一步：解析用户意图与参数

Claude 需要解析用户想要查询的”客户名称”和”日期范围”。
提取参数如下：
- `cli_name` (必填)：客户名称，如”三星”。支持模糊匹配。
- 日期条件（按需任选一）：
  - 当天：无需指定日期参数，脚本默认查询当天。
  - 指定日期：如”2026-03-01”，使用 `--date “2026-03-01”`。
  - 最近N天：如”近3天”，使用 `--days 3`。
  - 指定范围：如”2026-02-01 到 2026-02-28”，使用 `--start_date “2026-02-01”` 和 `--end_date “2026-02-28”`。

### 第二步：执行查询脚本

通过桥接层访问数据库，无直接 SQL：

```bash
# 示例：查询三星当天报价
python scripts/query_quote.py --cli_name “三星”

# 示例：查询华为近7天报价
python scripts/query_quote.py --cli_name “华为” --days 7
```

### 第三步：输出结果

脚本会将查询结果格式化输出。Claude 需要将控制台的输出结果整理后展示给用户（例如可以整理成更美观的 Markdown 表格），如果未查询到结果，则友好地告知用户”未找到符合条件的报价记录”。输出字段包括：日期、报价编号、客户、可提供型号、可提供品牌、可提供数量、DC (批号)、货期、备注。

## 注意事项

- **日期格式错误**：若用户提供的日期无法被解析（如异常的文本），尝试让用户提供标准的 `YYYY-MM-DD` 格式。
- **未提供客户名称**：`cli_name` 是必填项，若用户未提供，提示用户补充客户名称。
