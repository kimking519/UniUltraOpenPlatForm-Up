# 销售需求转报价 (Sale Trans Needs)

该 Skill 用于快速将需求单转换为初步的报价单，支持批量处理，并自动根据新规则生成以 `b` 开头加 5位数字自增的 `quote_id`。

---

## 前置条件

- 系统环境需可读写数据库 `uni_platform.db`。若未做特殊覆盖且未设置 `SALE_TRANS_NEEDS_DB_PATH` 环境变量，将通过默认配置定位数据库。
- Python 及必要的 `sqlite3` 内置模块需具备执行权限。

---

## 工作流程

### 第一步：获取要转报价的需求编号
系统在捕获到用户需要"转报价"某张需求单时，必须准确抓取需求单号参数。
- `--needs_ids` (必输)：需求单号关键词，支持模糊匹配，可以用空格传入多个。

### 第二步：预览并等待确认
使用 `--preview` 参数调用获取待转换单据的详情：
```bash
python scripts/trans_needs_20260303223256.py --preview --needs_ids x00001 0002
```
**注意：** Claude 必须将控制台打印出的查找到的单据信息整理后向用户确认：
> “我为您找到了需求单 x00001、x00002，确认要将它们转入报价单吗？”
如果未找到任何数据，请告知用户并停止工作流。

### 第三步：执行正式转换（用户同意后）
当用户回答“确认”、“是的”、“转入”等同意指令后，调用带 `--convert` 的执行脚本：
```bash
python scripts/trans_needs_20260303223256.py --convert --needs_ids x00001 0002
```
此时脚本会真正将选中的需求记录插入到 `uni_quote` 中，并为其生成如 `b00123` 的新报价单编号。同时它会将原需求表中的标记设为已处理（例如 `is_transferred=1`）。

### 第四步：反馈输出
Claude 将对脚本最终的转换结果打印信息整理后返回给用户：
> “✅ 转换成功！其中需求单 x00001 已转为报价单 b00123。”
