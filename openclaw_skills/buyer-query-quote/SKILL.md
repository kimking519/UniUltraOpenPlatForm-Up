# 采购端报价查询 (Buyer Query Quote)

该 Skill 用于根据指定的客户名及不同的选定时间段查询系统中存留的报价情况报表。该系统将综合呈现原需求与最新报价等 19 项详细对比明细表（含型号、成本、单利润与总利润等）。

---

## 前置条件

- 该模块为多表（如 `uni_quote`, `uni_needs`, `uni_cli`, `uni_supplier`）交叉联合计算查询脚本，确保系统内表结构字段未发生巨大位移变动。
- 保证预设的数据环境可通过配置文件 `db_config` 或通过 WSL 环境抛出的 `BUYER_QUERY_QUOTE_DB_PATH` 被主控正确加载访问。

---

## 工作流程

### 第一步：精确解析时间与客户名称参数
用户发起问询后，OpenClaw 会抽取对应的实体：
- `--cli_name` (必填)：提取其要求想要查询的公司（支持关键字提取用于模糊查询）。
- 日期时间变量提取：
  - 如果未包含任何时间指向，则为空，交给脚本按**当天**计算。
  - 用户说「几天前」、「近X天」：转换为 `--days`。
  - 用户指定范围「2026-03-01 到 2026-03-03」：提取为 `--start_date` 和 `--end_date`。

### 第二步：执行查询脚本
```bash
# 查询某客户某时间段记录
python scripts/query_quote_20260303230118.py --cli_name "LG" --start_date 2026-01-01 --end_date 2026-03-03

# 查询当天数据
python scripts/query_quote_20260303230118.py --cli_name "LG"
```

### 第三步：格式化输出响应报表
当执行完毕后，脚本直接在控制台排版打印出一个包含 19 个指标字段的长报表。由于字段数量非常庞大：
1. Claude 需将其直接转换为美观的 Markdown 大表格回复给用户进行查阅。
2. 表头必须遵从以下排列顺序：
   日期 | 报价编号 | 客户名称 | 需求编号 | 询价型号 | 报价型号 | 需求品牌 | 报价品牌 | 需求数量(pcs) | 报价数量(pcs) | 成本价 | 报价(RMB) | 报价(KWR) | 报价(USD) | 利润 | 总利润 | 供应商 | 批号(DC) | 交期(LT)
