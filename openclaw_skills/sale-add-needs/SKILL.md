# 销售添加与复制需求 (Sale Add Needs)

该 Skill 用于快速根据已有的需求编号复制并新建相同内容的单据，以提高业务录单效率。特别地，本工具在写入新记录时会确保生成的编号服从`x+5位顺序自增数字`的新规则。

---

## 前置条件

- 系统环境需可读写数据库 `uni_platform.db`。若未做特殊覆盖且未设置 `SALE_ADD_NEEDS_DB_PATH` 环境变量，将通过默认配置定位数据库。
- Python 及必要的 `sqlite3` 内置模块需具备执行权限。

---

## 工作流程

### 第一步：提取要复制的需求编号
系统在捕获到用户需要"复制"某张需求单时，必须准确抓取需求单号参数。
- `--needs_ids` (必输)：需求单号。支持模糊匹配（例如输入 `0001` 则匹配系统中包含 0001 的需求单，如 `x00001`）。可以输入多个供批量建立，使用空格隔开。

### 第二步：执行复制脚本
根据获取的参数调用对应的 Python 脚本：
```bash
# 示例 1: 单条精准/模糊复制
python scripts/add_needs_20260303222528.py --needs_ids x00001

# 示例 2: 批量多条复制
python scripts/add_needs_20260303222528.py --needs_ids 0001 0002
```

### 第三步：理解新编号机制并反馈输出
脚本内部对每个匹配成功的原有单据执行**深拷贝（除主键）**，并通过统计算出本表的最新 `x+5位数` 自增值予以重新赋号（例如最后一条是 `x00045`，则新单号必定为 `x00046`）。

Claude 将对脚本的打印信息执行处理后返回人类可读性语言：
- **存在成功项** : 明确反馈“需求 x00001 已成功复制为新单据 x00046”。
- **未查询到记录**: 反馈用户输入的原单号错误，请提示他们检查。
- **匹配出多条**: 脚本将会逐条全部执行克隆。

## 示例

### 输入示例
```
帮我把编号为 0001 的需求重新录一条。
```

### 输出示例
```
✅ 复制已完成。
根据您的要求，已成功将原需求记录（x00001）克隆为全新单据，其生成的新编号为：x00012。
```

---

## 注意事项 / 边缘情况

- **自动编号机制**：在首个单据生成前（当表里没有任何 `x` 开头的 5位数字的合法单据时），系统将使用基于 `x00001` 作为默认起始单号。
- **环境要求**：可以执行 WSL 环境下通过执行 `export SALE_ADD_NEEDS_DB_PATH="/path/to/db"` 进行热切换。
