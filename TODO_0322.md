# 📋 项目任务清单

> **执行规则：** 按顺序处理任务，完成一项后将 `[ ]` 改为 `[x]`，并填写分支名和完成备注。
> push 操作由人工测试确认后手动执行，Claude Code 不自动  push

---

## 🔀 Git 分支规范

### 每个任务的 Git 操作流程
```
1. 任务开始前：git checkout main分支
2. 任务完成后：git add . && git commit -m "fix: 描述"
3. 等待人工测试确认
4. 测试通过 → push 到github
5. 测试失败 → 打回，在原分支继续修改
```

### 任务状态说明
| 标记 | 含义 |
|------|------|
| `[ ]` | 待处理 |
| `[x]` | 已完成，等待测试 |
| `[~]` | 打回重做 |
| `[!]` | 紧急插队优先处理 |
| `[✅]` | 测试通过，已 merge |

## 📝 打回记录模板

当任务测试失败需打回时，在任务条目下补充：

```
- [~] 任务名称
  - 模块：
  - ❌ 第1次方案：做了什么 → 为什么无效
  - 🔍 测试反馈：具体报错或表现
  - 🎯 重做方向：根本原因在哪里，下次应该怎么改
```
