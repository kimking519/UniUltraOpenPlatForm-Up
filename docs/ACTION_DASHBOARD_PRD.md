# UniUltra 统一业务任务看板 (Unified Task Board) PRD v1.3

**版本**: 1.3 (核心业务驱动版)
**日期**: 2026-05-14
**设计理念**: 以“利润回收”和“客户留存”为核心，将零散的业务自动转化为“待处理”任务，由金总手动完成 [在处理 -> 待检验 -> 已完成] 的状态闭环。

---

## 1. 看板核心目标
1. **防流失**: 强制管理客户联系周期。
2. **促转化**: 严控报价时效与未下单跟进。
3. **保交付**: 监控未采购项与回款进度。
4. **强提醒**: 集中处理系统无法自动分类的重要事项。

---

## 2. 任务自动生成逻辑 (进入“待处理”列)

系统扫描数据库，符合以下条件的自动在“待处理”生成卡片：

### A. 客户联系 (CRM)
- **新名单**: `uni_contact` 中 `send_count = 0`。
- **周期预警**: 距离 `last_sent_at` 超过 7 天未联系的潜在客户。

### B. 报价周期 (Quote Management)
- **新询价**: 刚从邮件或录入生成的 `uni_quote`（状态：待处理）。
- **未下单跟进**: 报价已发出超过 48 小时，但 `is_transferred = '未转'` 的项，提醒金总去询问客户反馈。

### C. 采购与订单 (Ops)
- **待采购**: 已接客户订单 `uni_order`，但尚未生成采购单 `uni_buy` 的项目。
- **回款管理**: 已发货给客户，但 `is_paid = 0` 的订单。

### D. 综合预警 (Alerts)
- **重要提醒**: 包含退信处理、到期账期、或手动标记为“重要”的备注项。

---

## 3. 看板操作流程 (人工流转)

金总在看板页面对上述生成的卡片进行手动操作：

1. **[待处理] (Pending)**: 
   - 看到新任务（如：新询价）。
   - 开始介入操作（如：给供应商发消息）。
   - **手动点击** -> 进入 [在处理]。

2. **[在处理] (In Progress)**: 
   - 正在进行的任务（如：等待供应商报价、货在路上、正在写开发信）。
   - 完成关键步骤（如：拿到供应商价格了、货到了、收到水单了）。
   - **手动点击** -> 进入 [待检验]。

3. **[待检验] (Pending Inspection)**: 
   - 需要最后确认的动作（如：审利润率、验货 QC、确认银行到账）。
   - 确认无误。
   - **手动点击** -> 进入 [已完成]。

4. **[已完成] (Completed)**: 
   - 任务终结，归档记录。

---

## 4. 实施要求 (极简)
- **统一视图**: 所有 A/B/C/D 类任务混合显示，卡片通过标签（Label）区分类型（如：`询价`、`催款`、`联系`）。
- **一键切换**: 任务卡片下方提供三个按钮：`开始处理`、`去检验`、`完成`，点击即修改数据库 `status` 字段。
- **数据关联**: 点击卡片上的编号（如 Q2026051401），自动跳转至现有系统的编辑页面。
